2 Strathclyde Mechanical and Aerospace Research Toolbox for Mathematics
3 ------
4 
5 `SMART` is a collection of toolboxes developed and maintained since 2015 by the department of Mechanical and Aerospace Engineer of Strathclyde University. `SMART-MATH` in particular, is a collection of mathematical core routines.
6 
7 `SMART` is aimed at making code development inside the team more efficient, reusable and easy maintainabile.
8 
9 `SMART` is CMake-based C++ project. It can be used as the basis for new projects.

40 Installation
41 ------
42 
43 Run the following commands to download, build, and install this project.
44 
45  git clone https://github.com/space-art/smart-math.git
46  (Introcude your username and password)
47  cd smart-math
48  mkdir build && cd build
49  cmake .. && make
50 
51 To install the header files, run the following from within the `build` directory:
52 
53  make install
54 
55 Note that dependencies are installed by fetching them online, in case they cannot be detected on your local system. If the build process fails, check the error log given. Typically, building fails due to timeout. Simply run the `cmake --build .` command once more.

57 Build options
58 -------------
59 
60 You can pass the following, general command-line options when running CMake:
61 
62  - `-DCMAKE_INSTALL_PREFIX[=$install_dir]`: set path prefix for install script (`make install`); if not set, defaults to usual locations
63  - `-DBUILD_SHARED_LIBS=[on|off (default)]`: build shared libraries instead of static
64  - `-DBUILD_MAIN[=on|off (default)]`: build the main-function
65  - `-DBUILD_DOCS[=ON|OFF (default)]`: build the [Doxygen](http://www.doxygen.org "Doxygen homepage") documentation ([LaTeX](http://www.latex-project.org/) must be installed with `amsmath` package)
66  - `-DBUILD_TESTS[=ON|OFF (default)]`: build tests (execute tests from build-directory using `ctest -V`)
67  - `-DBUILD_DEPENDENCIES[=ON|OFF (default)]`: force local build of dependencies, instead of first searching system-wide using `find_package()`
68 
69 The following command is conditional and can only be set if `BUILD_TESTS = ON`:
70 
71 
72 requires [GCC](https://gcc.gnu.org/) compiler; execute coverage analysis from build-directory using `make coverage`)
73 
74 Pass these options either directly to the `cmake ..` build command or run `ccmake ..` instead to bring up the interface that can be used to toggle options.
